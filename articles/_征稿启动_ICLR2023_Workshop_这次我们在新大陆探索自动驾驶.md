【征稿启动】ICLR2023 Workshop：这次我们在新大陆探索自动驾驶
======================================

原创 OpenDriveLab OpenDataLab 2023-02-08 16:55 上海

> 原文地址: [https://mp.weixin.qq.com/s/CPrxsBYM5pfvENB4py1QGw](https://mp.weixin.qq.com/s/CPrxsBYM5pfvENB4py1QGw)

**深度学习顶级学术会议 ICLR 2023** 即将举办，欢迎投稿**【自动驾驶技术中的场景表征】Workshop**，与全球顶尖研究者共话自动驾驶汽车研究及未来发展。

  

![](https://mmbiz.qpic.cn/mmbiz_png/7yjDpC9UfD7ViaLWVRvic3vEhu3bG6ssbtD7sKr3BO3LpS0ia8WEXrOVe78ZFTDcgficEjpAiby2EKK0SaLywnxDxag/640?wx_fmt=png)

********会议简介********

![](https://mmbiz.qpic.cn/mmbiz_png/7yjDpC9UfD7YNNFaRmQrHELaFarjLQYtdXDAR1GW90GlAdYdkficbx7jyMtNRArKDUIFLAgzWvQKnFVmW7zvR1w/640?wx_fmt=png)

国际学习表征会议 ICLR（International Conference on Learning Representations）作为深度学习领域的顶级会议，在谷歌发布的人工智能领域“top publication”排名中位列前十。ICLR 将于 2023 年5月1-5日在卢旺达举办第十一届会议，**这是机器学习领域顶会首次在非洲国家举办**，这将会极大促进非洲国家科技发展及民众对前沿科技的认知，进而带来广泛而深远的国际影响力。

  

1月21日 ICLR 官方公布了Proposal的最终收录结果，各大 Workshop及挑战赛已相继启动报名，其中**【自动驾驶技术中的场景表征】**是由上海人工智能实验室自动驾驶 OpenDriveLab 团队筹备举办，诸多国内外学者参与的顶会 Workshop，致力于推动实际应用场景下的自动驾驶技术的发展。

![](https://mmbiz.qpic.cn/mmbiz_png/7yjDpC9UfD6kOkDjD84MS44PRQY2hSBgE6AnVDzGJg44v6oY04l8pJT0lAzVheT07Bv1Wzc2G7SX9VLG38ZRxg/640?wx_fmt=png)

论坛官方网站：

http://opendrivelab.com/sr4ad/iclr23

  

**论坛介****绍**

本论坛是 **OpenDriveLab** 关于端到端自动驾驶汽车的系列论坛之一，致力于将对现实世界具有重要影响的机器学习算法研究应用到自动驾驶技术领域。在当前的自动驾驶研究领域，将多个基于机器学习的算法模块直接堆叠的技术手段已经取得了工业应用上的成功；然而在多个算法模块的集成策略和不同模块间数据的中间表示（representation）这些研究领域内仍有广袤的探索空间。

  

因此，OpenDriveLab 将筹备举办 ICLR 2023 研讨会：**【自动驾驶技术中的场景表征】**，我们邀请广大研究者就以下**五类话题**展开研讨工作，并邀请六位演讲嘉宾。我们希望本次活动能够赋能下一代自动驾驶汽车的发展，欢迎大家积极参与。

●关于自动驾驶汽车的环境感知、预测、路径规划和仿真等领域的表征学习研究；

● 关于传统自动驾驶汽车的不同子模块之间的信息交互方法研究（如联合的感知预测，端到端的自动驾驶等）；

● 关于使用基于机器学习或统计学习的方法促进自动驾驶汽车的安全性 safety/算法可解释性 interpretability/算法泛化性 generalization的研究；

● 用于确立自动驾驶机器学习算法标杆基准的驾驶环境/数据集；

● 关于未来新一代自动驾驶技术的新视角前瞻研究。

**工作征集**

为了促进内容的丰富性，本论坛征集的工作形式包括：（1）**博客文章**

（2）**Github仓库**（3）**PDF格式的文章**（2-8页）。提交到本论坛的工作将被分到如下两个方向：

_********No.1********_ ********科研洞察方向********

● 作者需要提交关于**他人工作**的分析或复现文档 / 代码 / 博客。在这个方向中作者应作为该工作的报告员或评论员；

●作者必须声明自身和提交到本论坛的文章（和原文章的作者）之间的利益冲突（包括正面的或负面的关系，如合作关系、相同的工作机构、比赛中的竞争关系等）；

● 提交的新工作中，相较于引用的原论文有**显著价值的附加**内容将被着重审阅；

● 所有的分析与评论必须以准确和清晰的论述作为支撑。

  

_****No.2****_  **原创工作**

● 作者需要提交没有被发表的原创工作；

● 委员会将审阅所提交的工作中的观点是否均具有可信的实证支撑。

  

**重要日期**

工作提交截止日期：**2023年2月15日23:59**（按全球各地当地时间）

收录决定通知日期：**2023年3月3日**

提交链接：

https://openreview.net/group?id=ICLR.cc/2023/Workshop/SR4AD

  

**格式要求**

● 对于博客博文，请提交包含所有待审阅内容的 **URL** 。对于博客方向并没有对文章字数和格式的强制要求。请作者参考 **ICLR 2022 博客博文网站**以获取样例；

● 对于 Github repo，请提交一个说明如何运行代码的 **README.md 文件的 URL**；

● 对于 PDF 格式的文章，请作者使用 **ICLR的撰写模板**。提交的文章可以是摘要的扩展形式（2-4页）或完整文章格式（4-8页）。参考文献和附录不受页数限制。

**内容审阅细则**

● 所有的工作可使用 **OpenReview** 进行提交。提交链接将于近期在网站上公布，请关注网站信息；  

● 评审过程将采用**单盲评审**；

●提交的原始工作和评审材料将不会对外公布，只有接收的工作将会被对外公布；

●本论坛属于**非归档类**研讨分会场，不具有官方论文集。以 PDF 格式提交的论文工作也可以在后续或同时提交到其他分会场。

●本论坛所有接收的工作都会被以墙报的形式展示；

●对于每篇被接收的工作，至少一位作者需要在 ICLR 2023注册并参加墙报展示与讨论环节（允许远程形式参加）；

●所有被接收的工作都将被收录在本论坛的官方网站（对于被接收的工作，作者也可以指明是否主动弃权）。

如有任何问题，请联系邮箱：workshop-e2e-ad@googlegroups.com 并在标题附上前缀 \[ICLR 2023 SR4AD\]。
